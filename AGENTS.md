# AGENTS.md

## Dev environment tips
- Install dependencies with `npm install` before running scaffolds.
- Use `npm run dev` for the interactive TypeScript session that powers local experimentation.
- Run `npm run build` to refresh the CommonJS bundle in `dist/` before shipping changes.
- Store generated artefacts in `.context/` so reruns stay deterministic.

## Testing instructions
- Execute `npm run test` to run the Jest suite.
- Append `-- --watch` while iterating on a failing spec.
- Trigger `npm run build && npm run test` before opening a PR to mimic CI.
- Add or update tests alongside any generator or CLI changes.

## PR instructions
- Follow Conventional Commits (for example, `feat(scaffolding): add doc links`).
- Cross-link new scaffolds in `docs/README.md` and `agents/README.md` so future agents can find them.
- Attach sample CLI output or generated markdown when behaviour shifts.
- Confirm the built artefacts in `dist/` match the new source changes.

## Repository map
- `apps/` — Contains 4 independent applications: `backend-express` (Express.js REST API), `backend-nestjs` (NestJS REST API), `frontend-nextjs` (Next.js React frontend), `frontend-svelte` (SvelteKit Svelte 5 frontend). Edit when adding features, fixing bugs, or modifying application logic.
- `packages/` — Shared workspace packages. Currently contains `shared/` with TypeScript types, DTOs, constants, and seed data used by all apps. Edit when adding new data models, constants, or shared utilities. Always rebuild after changes (`npm run build -w packages/shared`).
- `package.json` — Root workspace manifest defining npm workspaces (`packages/*`, `apps/*`) and cross-app scripts (`dev:express`, `dev:nestjs`, `dev:svelte`, `dev:nextjs`, `build`, `seed:*`). Edit when adding new workspace scripts or dependencies.
- `package-lock.json` — Auto-generated npm lockfile. Never edit manually — regenerated by `npm install`.
- `tsconfig.base.json` — Base TypeScript configuration (ES2022 target, strict mode, bundler resolution). Edit when changing compiler options that apply to all workspaces.
